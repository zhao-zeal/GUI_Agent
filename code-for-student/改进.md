**参考github**
AgentCPM-GUI：强调移动端 GUI grounding、中文 App 操作、精简 action JSON 和 “think-before-acting”。它的启发是：不要只靠 VLM 原始坐标，要把动作空间压缩成稳定格式，并让模型输出更可解析的动作。
Mobile-Agent：是比较成熟的移动端 GUI Agent 系列，方向上更像“任务规划 + 视觉定位 + 动作执行”的组合，而不是单步 VLM。
UI-Voyager：核心是从失败经验中自进化，这和我们现在做的 failure report → 修规则 → 再抽象非常接近。
MobileRL：思路是通过在线 agentic RL 提升移动 GUI Agent，但对我们比赛而言，没法训练模型，所以更现实的是把“经验”变成状态机和 UI anchor。
GELab-Zero：强调本地可部署、移动 GUI Agent 评测和定位能力，说明移动端 GUI Agent 的关键仍是 grounding + 任务状态。
